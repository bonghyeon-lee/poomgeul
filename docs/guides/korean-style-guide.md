## 적용 범위

- **논문 한정** (Phase 1, 영→한).
- 문학 번역은 별도 규정(Phase 2). 원작의 문체 보존 — 존댓말/반말·구어/문어는 작품별 판단.
- AI 초벌 프롬프트의 system instruction·에디터 힌트·리뷰 체크리스트가 모두 이 문서를 참조합니다.

## 1. 문체·톤

- **기본 문체:** 평서문 + `-이다/하다`체 (음슴체/문어체).
  - "~합니다/입니다"체 **아님**. 학술 논문 번역의 관행을 따름.
- **1인칭 처리:**
  - 원문 "We propose ..." → **"우리는 ~를 제안한다"** 보존.
  - "저자는"으로 치환 **금지** (원저자 의도 왜곡).
- 문학 번역은 이 섹션이 적용되지 않음(별도 규정).

## 2. 전문 용어 표기

용어 일관성은 M2 글로벌 용어집 전까지 **본 가이드의 하드 규칙 + 번역본별 용어집**으로 담보합니다.

| 상황 | 규칙 | 예시 |
|---|---|---|
| 한국어에서 이미 정착 | 한글 표기 우선 | `backpropagation` → **역전파** |
| 한국어 번역이 혼재 | 첫 등장에 `한글(영어)` 병기, 이후 한글만 | `transformer` → **트랜스포머(transformer)** → 트랜스포머 |
| 정착 전·논쟁 중 | 원어 유지 + 역주 각주 | `embedding` → **embedding** (역주: 임베딩으로도 번역) |
| 고유명사·모델 이름 | 원어 유지 | GPT-4, LLaMA |
| 수식·코드 | 번역 안 함 | `\alpha_t`, `softmax(x)` |

## 3. 띄어쓰기·문장부호

- **띄어쓰기:** 국립국어원 표준 기준.
  - 예외: 기술 용어 조합("딥러닝 모델" vs "딥러닝모델")은 한글 표기 유지 쪽으로.
- **문장부호:** 한국어 쉼표·마침표.
  - 영어 em dash(`—`)는 한국어 쉼표 또는 `―`로 치환.
- **따옴표:** `"..."` (큰따옴표), 중첩은 `'...'` (작은따옴표).
  - 영문 스마트쿼트 `" "` **금지**.
- **숫자·단위:** 아라비아 숫자 + 한국어 단위.
  - "10%"는 그대로. "ten percent"는 "10%"로.

## 4. 보존 원칙 (번역하지 않는 것)

다음은 번역 대상이 아니므로 **원문 그대로** 보존합니다.

- 인라인 수식 `$...$`, 블록 수식 `\begin{equation}...\end{equation}`.
- 코드 블록.
- 인라인 citation (예: `[3]`, `Wang et al., 2023`).
- 테이블·그림 참조 (예: `Table 1`, `Figure 2`, `Section 3.1`).
- 저자·기관·모델·제품 고유명사 (GPT-4, Claude, Gemini, Llama, arXiv 등).

## 5. 번역 노트·역주

- 번역 노트는 **세그먼트에 첨부되는 선택적 필드** (`Metadata.Note` — [data-model.md](../architecture/data-model.md)).
- M0는 테이블 존재 + 기본 표시만. 인라인 각주 UI는 M2.
- 권장 사용처:
  - 번역 불가 단어 (pun, 문화적 맥락).
  - 원문 오류·오타로 의심되는 부분.
  - 복수 번역안이 합리적인 경우.

## 6. 체크리스트 (리뷰어용)

제안을 승인하기 전 다음을 확인합니다.

- [ ] 문체가 `-이다/하다`체인가?
- [ ] "저자는"으로 `We`를 치환하지 않았는가?
- [ ] 인라인 citation·Figure/Table 참조가 원문 그대로인가?
- [ ] 수식·코드가 **편집되지 않았는가?** (잠금 위반 없음)
- [ ] 정착된 한글 용어가 일관되게 사용되었는가?
- [ ] 문장부호가 한국어 규범에 맞는가?
- [ ] 용어 첫 등장에 필요한 병기가 되어 있는가?

## 7. AI 초벌 프롬프트와의 연결

본 규정은 M0 단일 고정 프롬프트([prompts/translate.en-ko.v1.md](../../prompts/translate.en-ko.v1.md))의 system instruction에 그대로 박힙니다. 별도 설정 UI 없이, 운영자가 프롬프트 레포를 Git 버전 관리하는 방식.

M2에서 **번역본별 커스텀 프롬프트** 허용 시, 위 규정을 default로 두고 리드가 덮어쓸 수 있게 합니다.

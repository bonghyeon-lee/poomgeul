# 라이선스 정책

![phase](https://img.shields.io/badge/phase-M0%2B-green)

기획서 §10 발췌. poomgeul에서 저작권은 **부가 기능이 아니라 하드 제약**입니다.

## 허용 라이선스 (M0)

| 라이선스 | 원문 등록 | 번역본 | 번역본 라이선스 |
|---|---|---|---|
| **CC BY** | ✅ | ✅ | 자유 선택 (attribution 필수) |
| **CC BY-SA** | ✅ | ✅ | CC BY-SA **자동 고정** (ShareAlike) |
| **퍼블릭 도메인 (PD)** | ✅ | ✅ | 자유 선택 |
| CC BY-NC | 단계적 허용 검토 | — | 비상업적 한정 |
| CC BY-ND / CC BY-NC-ND | ❌ 번역 자체 금지 | — | — |
| 독점 저작권 | ❌ | — | — |

## 플랫폼 레벨 자동화

M0 시점부터 아래 자동화를 포함합니다. 법적 리스크를 사실상 0에 수렴시키는 것이 목표.

- 원문 등록 시 **라이선스 필수 선택 + 출처 URL 검증**.
- arXiv API / Crossref API / DOAJ로 **라이선스 메타데이터 자동 조회·검증**.
- CC BY-ND / CC BY-NC-ND는 **등록 자체를 차단**.
- CC BY-SA 원문 → 번역본 라이선스를 **CC BY-SA로 자동 고정**.
- 모든 페이지에 **Attribution 블록 자동 생성** ([attribution.md](attribution.md)).
- **DMCA / 저작권 신고 접수·철회 프로세스** 미리 마련.

## 주의 지점

- **라이선스 표기의 진위:** 저자 웹사이트가 아니라 arXiv API / 저널 메타데이터가 기준.
- **논문 내 제3자 자료** (그림·표·인용 이미지)는 별도 저작권 가능 → **텍스트만 번역, 이미지는 원문 링크로 참조**.
- **버전별 라이선스 차이** — arXiv v1 vs v2는 별도 `source_version`.
- **프리프린트 vs 저널판:** arXiv CC BY 프리프린트만 사용, 저널판 PDF 사용 금지.

## 퍼블릭 도메인 (Phase 2 별도 고려)

- 국가별 보호 기간이 다름 (한국 70년, 미국·EU 70년, 멕시코 100년).
- 원작이 PD여도 **기존 번역본은 역자 기준으로 별도 보호 가능**.
- 플랫폼은 "원문 저작권 상태"와 "기존 번역본 저작권 상태"를 **분리해서 다룸**.

## 기여자 개인정보 & 데이터 보호 (§10.5)

저작권과 별개로, 기여자의 개인정보는 플랫폼 신뢰의 또 다른 축입니다. GDPR·한국 개인정보보호법 준수 전제.

- **수집 최소 원칙:** 이메일(인증용) + 선택적 표시 이름 + OAuth 연동 ID(GitHub/ORCID)만 필수. 실명·주소·결제정보는 관리형 호스팅 유료 플랜에서만.
- **공개 범위 선택:** 프로필에서 (1) 이메일 비공개, (2) 표시 이름만 노출, (3) 기여 이력 공개/비공개를 각각 제어. 기본값은 "이름 공개 + 이메일 비공개".
- **삭제권:** 계정 삭제 시 개인 식별 정보는 제거, **번역 기여 자체는 오픈 라이선스로 공개된 상태이므로 유지**(GitHub가 계정 삭제 시 커밋을 유지하는 것과 동일 원칙). 이 정책은 가입 시 명시.
- **LLM 프롬프트 로그:**
  - AI 초벌 시 원문+프롬프트가 외부 LLM API로 전송됨.
  - 기여자의 교정 내용이 LLM 제공사에 전송되지 않도록 경계 분리.
  - 학습 opt-out 명시, 가능하면 Zero Data Retention 옵션 사용.
- **무료 tier 데이터 사용 경계:** Gemini Free tier 등은 ToS상 프롬프트가 학습 데이터에 사용될 수 있음. **절대 전송 금지** 범주:
  1. 기여자가 쓴 교정·코멘트·제안 본문
  2. 아직 공개되지 않은 저자 직접 등록 원문(유형 A)
  3. 계정 식별자·이메일을 포함한 프롬프트

  → 이 범주는 유료 tier(ZDR 적용) 또는 BYO Key 경로로만 처리. 구현 체크리스트는 [guides/llm-integration.md §데이터 경계](../guides/llm-integration.md) 참조.
- **쿠키·분석:** 초기엔 행동 분석 도구 없이 서버 로그만. Plausible·Umami 같은 프라이버시 친화 분석은 후순위.
- **데이터 레지던시:** EU 사용자 비중이 일정 이상 되면 EU 리전 호스팅 검토 (Phase 2 이후).
